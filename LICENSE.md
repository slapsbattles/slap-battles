## 🔧 Common Issues & Fixes

**❓ The GUI doesn't appear after executing**
> Make sure your executor is properly attached to Roblox *before* executing. Re-attach and try again. If using mobile, ensure the executor has overlay permission granted.

**❓ Kill aura stops working mid-game**
> This usually happens after a server-side game update. Check this repository for a patch. As a temporary fix, toggle kill aura off and back on from the GUI.

**❓ Badge unlock is stuck or incomplete**
> Some badges require you to be in a specific in-game state. Make sure you're in an active arena (not the lobby) before running the badge module. If it stalls, rejoin a fresh server and re-execute.

**❓ Anti-void isn't catching my character**
> Increase the anti-void trigger threshold in the GUI settings. The default is tuned for standard knockback; some gloves with extreme force require a higher sensitivity setting.

**❓ Script detected / account warned**
> Stop using the script immediately and allow 72 hours before logging back in. Review the "Tips for Safer Use" section above and consider switching to an alt account going forward.

**❓ Executor crashes when script loads**
> Your executor version may be outdated. Update to the latest release of your executor. KRNL users should ensure the bootstrapper is re-run after any Windows update.

**❓ All gloves feature doesn't equip the admin glove**
> Admin glove handling requires a separate module. Enable **Admin Glove Mode** in the Advanced Settings tab of the GUI — this activates the **admin glove slap battles script** component, which handles the glove's unique server-side logic separately.
